## Best Practices

- **Check Schema First**: Use `get_table_schema` to understand structure
- **Test Filters**: Use `query_records` to test filters before updates/deletes
- **Use Appropriate Filters**: Be specific to avoid unintended changes
- **Batch Operations**: Group related operations together
- **Verify Changes**: Query data after modifications to confirm results

## Safety Features

- **SQL Injection Protection**: All tools use parameterized queries
- **Required Filters**: Update and delete operations require filter conditions
- **Confirmation Prompts**: Destructive operations need explicit confirmation
- **Limit Safeguards**: Maximum record limits prevent accidental mass operations
