## Tools Reference

| Tool | Risk Level | Confirmation | Purpose |
|------|------------|--------------|---------|
| `list_tables` | Safe | No | Discover database structure |
| `get_table_schema` | Safe | No | Understand table definitions |
| `create_table` | Medium | No | Build new tables |
| `alter_table` | Medium | No | Modify table structure |
| `alter_column` | Medium | No | Modify column properties |
| `rename_table` | Medium | No | Rename tables safely |
| `drop_table` | High | Yes | Remove entire tables |
| `add_index` | Low | No | Create performance indexes |
| `drop_index` | Low | No | Remove unused indexes |
| `add_constraint` | Medium | No | Enforce data integrity |

## Best Practices

- **Start with Discovery**: Use `list_tables` and `get_table_schema` first
- **Plan Changes**: Design modifications carefully before execution
- **Test First**: Verify changes in development environments
- **Use Confirmations**: Always confirm destructive operations
- **Index Strategy**: Create indexes for frequently queried columns
