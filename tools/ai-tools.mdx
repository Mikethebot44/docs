## Key Features

### Research Agent
- **Two Modes**: Quick (30 seconds) for fast facts, Deep (2-5 minutes) for analysis
- **Structured Results**: Organized findings with source citations and relevance scores
- **Current Information**: Access to real-time web data and recent developments
- **Topic Analysis**: Compare technologies, market research, best practices

### Semantic Search
- **Natural Language**: Search using conversational queries instead of keywords
- **Multi-Source**: Search repositories, documentation, and database tables
- **Similarity Scoring**: Results ranked by relevance with confidence scores
- **Metadata Rich**: Detailed information about search results and context

### Vector Index Management
- **Multiple Sources**: Index repositories, documentation, and database schemas
- **Lifecycle Management**: Create, monitor, update, and delete indexes
- **Performance Monitoring**: Track index status, size, and search performance
- **Batch Operations**: List and manage multiple indexes efficiently

## Configuration

### Required Setup
```bash
# Tablr API Configuration
TABLR_API_URL=https://www.tablr.dev
TABLR_API_KEY=your_api_key_here

# Performance Tuning (Optional)
VECTOR_API_TIMEOUT=30000
VECTOR_API_MAX_RETRIES=3
```

## Best Practices

- **Specific Queries**: Use focused, specific research queries for better results
- **Appropriate Thresholds**: Balance relevance vs completeness in search results
- **Index Strategy**: Create indexes for frequently searched content
- **API Key Security**: Store API keys in environment variables, never in code

## Integration Patterns

### Research-Driven Development
1. **Research** current best practices and technologies
2. **Deep Dive** into specific implementations and comparisons  
3. **Search** existing codebase for similar patterns and solutions

### Documentation Enhancement
1. **Index** documentation sources for semantic search
2. **Search** for specific examples and configuration patterns
3. **Monitor** index status to keep content current
