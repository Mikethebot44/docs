## Prerequisites

Before you start, make sure you have:

- **Node.js 18.0.0+** installed on your system
- **PostgreSQL database** with connection access
- **MCP-compatible AI assistant** (Claude Desktop, Cursor, etc.)

## Step 1: Prepare Your PostgreSQL Database

Ensure you have your PostgreSQL connection details:
- **Host** (e.g., `localhost` or `your-db-host.com`)
- **Port** (default: `5432`)
- **Database name**
- **Username** and **Password**
- **SSL configuration** (if required)

<Note>
You'll need a user account with sufficient privileges to create, modify, and query tables.
</Note>

## Step 2: Install Tablr MCP

Install the Tablr MCP server globally:

```bash
npm install -g tablr-mcp
```

## Step 3: Configure Environment Variables

Create a `.env` file in your project directory:

```env
TABLR_API_KEY=
PG_CONNECTION_STRING=postgresql://postgres.bivecvumemthweyubvja:[YOUR-PASSWORD-HERE]@aws-0-ap-southeast-2.pooler.supabase.com:6543/postgres
SUPABASE_URL=
SUPABASE_SERVICE_ROLE_KEY=
```

## Step 4: Configure Your AI Assistant

Add this configuration to your Claude Desktop/Cursor/Windsurf/VS Code settings:

```json
{
  "mcpServers": {
    "tablr": {
      "command": "tablr-mcp",
      "args": ["absolute/path/to/your/.env"]
    }
  }
}
```
Note: do not use backslashes for windows file paths

## Step 5: Start Using Tablr

Restart your AI assistant and start managing your database with natural language:

### Basic Commands

```
"Show me all tables in my database"
"What's the schema for the users table?"
"Create a new table called 'posts' with title and content columns"
"Find all users created in the last week"
"Insert a new user with email john@example.com"
```

### Advanced Operations

```
"Add an index on the email column for better performance"
"Update all inactive users to set their status to archived"
"Execute this SQL: SELECT COUNT(*) FROM orders WHERE status = 'pending'"
"Create a backup of the important_data table"
```

## Verification

To verify everything is working:

1. **Test Connection**: Ask your AI assistant "List all tables"
2. **Test Operations**: Try creating a simple table or querying existing data

## Common Issues

<AccordionGroup>
  <Accordion icon="exclamation-triangle" title="Authentication Failed">
    **Problem**: "Authentication failed" or "Connection refused"
    
    **Solution**: 
    - Verify your PostgreSQL connection details
    - Check that PostgreSQL server is running
    - Ensure network connectivity to database host
    - Verify username and password are correct
  </Accordion>
  
  <Accordion icon="database" title="No Tables Found">
    **Problem**: "No tables found in database"
    
    **Solution**:
    - Verify your PostgreSQL database contains tables
    - Check that tables exist in the target schema
    - Ensure user has proper permissions to view tables
  </Accordion>
  
  <Accordion icon="plug" title="Connection Issues">
    **Problem**: MCP server won't start
    
    **Solution**:
    - Verify Node.js version (18.0.0+)
    - Check file paths in MCP configuration
    - Restart your AI assistant after configuration changes
  </Accordion>
</AccordionGroup>
