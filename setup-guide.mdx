## Overview

This guide will walk you through setting up Tablr MCP for semantic search and content indexing. You'll install the MCP server, configure your API access, and integrate with your AI assistant.

## Prerequisites

Before starting, ensure you have:

### Required Software

**Node.js (Version 18.0.0 or Higher)**
- Download from [nodejs.org](https://nodejs.org/)
- Choose the LTS (Long Term Support) version

**Verification:**
```bash
node --version  # Should show v18.x.x or higher
npm --version   # Should show 8.x.x or higher
```

**Git (Version Control)**
- Download from [git-scm.com](https://git-scm.com/)

**Verification:**
```bash
git --version  # Should show git version 2.x.x
```

### API Access Requirements

**Tablr API Key**
- Active Tablr account at [tablr.dev](https://tablr.dev)
- Valid API key with vector database permissions
- Sufficient credits for indexing and search operations

<Note>
You can sign up for a free Tablr account to get started with vector database operations. Paid plans offer higher usage limits and additional features.
</Note>

## Installation

### Method 1: Global Installation (Recommended)

Install Tablr MCP globally for easy access:

```bash
npm install -g tablr-mcp
```

### Method 2: Local Installation

Install locally in your project directory:

```bash
npm install tablr-mcp
# Use with npx
npx tablr-mcp
```

### Verification

Test your installation:

```bash
tablr-mcp --help
# Expected output: Tablr MCP Server - Vector database operations
```

## Configuration

### Environment Setup

Create a `.env` file in your project root directory:

```bash
# Create .env file
touch .env  # Unix/macOS/Linux
# or
type nul > .env  # Windows
```

### API Key Configuration

Add your Tablr API key to the `.env` file:

```env
# Required: Tablr API key
TABLR_API_KEY=your_tablr_api_key_here

# Optional: API endpoint (defaults to production)
TABLR_API_URL=https://www.tablr.dev
```

<Warning>
**Security**: Never commit your `.env` file to version control. Add it to your `.gitignore` file.
</Warning>

### Test API Connection

Verify your configuration:

```bash
# Start the server to test API connection
npm start
```

You should see:
```
Starting Tablr MCP Server...
Initializing vector database client...
Successfully connected to Tablr API
MCP Server connected via stdio
Server started successfully
```

## AI Assistant Integration

### Claude Desktop Configuration

Add this configuration to your Claude Desktop settings:

```json
{
  "mcpServers": {
    "tablr": {
      "command": "tablr-mcp",
      "args": ["/absolute/path/to/your/.env"]
    }
  }
}
```

### Other MCP Clients

**Cursor IDE:**
```json
{
  "mcp.servers": {
    "tablr": {
      "command": "tablr-mcp",
      "args": ["/absolute/path/to/your/.env"]
    }
  }
}
```

**VS Code with Continue:**
```json
{
  "mcpServers": [
    {
      "name": "tablr",
      "command": "tablr-mcp",
      "args": ["/absolute/path/to/your/.env"]
    }
  ]
}
```

<Note>
Replace `/absolute/path/to/your/.env` with the actual absolute path to your environment file. Do not use relative paths or backslashes on Windows.
</Note>

### Common Issues

<AccordionGroup>
  <Accordion icon="exclamation-triangle" title="Connection Failed">
    **Problem**: "Authentication failed" or "Connection refused"
    
    **Solutions**:
    - Verify PostgreSQL server is running
    - Check connection details in `.env` file
    - Ensure network connectivity to database host
    - Verify username and password are correct
    - Check firewall settings
  </Accordion>
  
  <Accordion icon="database" title="No Tables Found">
    **Problem**: "No tables found in database"
    
    **Solutions**:
    - Verify database contains tables
    - Check user has permission to view tables
    - Ensure connecting to correct database/schema
  </Accordion>
  
  <Accordion icon="plug" title="MCP Server Issues">
    **Problem**: MCP server won't start
    
    **Solutions**:
    - Check Node.js version (18.0.0+)
    - Verify file paths in MCP configuration
    - Restart AI assistant after configuration changes
    - Check for port conflicts
  </Accordion>
</AccordionGroup>

## Security Best Practices

### Credential Management

```bash .gitignore
# Environment variables
.env
.env.local
.env.production

# API credentials
*.key
*.pem
config/secrets.json
```

### API Key Security

Protect your Tablr API key:

- **Never commit** API keys to version control
- **Rotate keys** regularly in the Tablr dashboard
- **Use environment variables** for different environments
- **Monitor usage** through the Tablr dashboard
- **Set usage limits** to prevent unexpected charges

### Access Control

For production environments:
- Use separate API keys for different environments
- Monitor API usage and costs
- Implement rate limiting if needed
- Regularly review indexed content and permissions

## Advanced Configuration

### Environment-Specific Settings

**Development:**
```env
TABLR_API_KEY=dev_api_key_here
TABLR_API_URL=https://dev.tablr.dev
DEBUG=true
```

**Production:**
```env
TABLR_API_KEY=prod_api_key_here
TABLR_API_URL=https://www.tablr.dev
NODE_ENV=production
```

### Performance Tuning

```env
# API request settings
VECTOR_API_TIMEOUT=30000
VECTOR_API_MAX_RETRIES=3

# Search result limits
VECTOR_DEFAULT_RESULTS=10
VECTOR_MAX_RESULTS=100
```
